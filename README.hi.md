<p align="center"><b>A diagnostic adversarial game for frontier LLMs — a measurement instrument that happens to be fun.</b></p>

एक क्लाउड सत्र (**डिजाइनर**) वास्तविक, वर्तमान में देखे गए क्षमता अंतरों को लक्षित करते हुए पहेलियाँ बनाता है। एक अन्य (**सॉल्वर**) उन्हें हल करने का प्रयास करता है। नीति द्वारा लागू किया गया कर्नेल मध्यस्थता करता है, एक छिपे हुए ओरेकल के विरुद्ध स्कोर करता है, और `लैब → एरीना → रिग्रेशन` जीवनचक्र के माध्यम से एक सूची तैयार करता है। पहेलियाँ अनुभवजन्य संकेतों पर आधारित होती हैं - वास्तविक GitHub मुद्दे, अकादमिक साहित्य, क्षेत्र में देखी गई विफलताएँ - न कि कृत्रिम रूप से बनाई गई।

## यह इसे अलग कैसे बनाता है

- **क्षमता, न कि "धोखा"।** एआई क्रूसिबल *सुरुचि* और *नवीनता* (पुरस्कृत) को *उत्तर-बypास* (दंडित) से अलग करता है। रचनात्मक सोच एक मापने योग्य क्षमता है, दंड देने लायक कोई बुराई नहीं।
- **उपकरण स्वयं को मापता है।** प्रॉम्प्ट फ्रेमिंग एक प्रथम श्रेणी का मापने योग्य पहलू है - कर्नेल `तटस्थ` / `स्व-संदर्भित` / `सामाजिक स्थिति` फ्रेमिंग के तहत समान पहेली चलाता है और अपने स्वयं के प्रॉम्प्ट प्रभाव को एक नैदानिक ​​रूप में रिपोर्ट करता है।
- **एक सीलबंद माप सीमा।** प्रेरणा और माप कभी भी एक ही संदर्भ विंडो साझा नहीं करते हैं; छिपे हुए ओरेकल का मूल्यांकन एक अलग मॉडल परिवार द्वारा किया जाता है, जिसमें एजेंट की तर्क प्रक्रिया छिपी रहती है। मॉडल वह गेम नहीं खेल सकता जिसे वह समझ नहीं सकता।
- **संगति के माध्यम से विश्वसनीयता।** `पास^k` (सभी *k* स्वतंत्र परीक्षण सफल), विल्सन अंतराल और क्रॉस-परिवार जज पैनल - वितरण रिपोर्ट करने के लिए बनाए गए, न कि बिंदु अनुमानों के लिए।
- **एक जीवंत, टिकाऊ सूची।** रन एक घटना-आधारित, हैश-श्रृंखलाबद्ध लॉग में सत्रों में जमा होते हैं (सत्य का स्रोत); टियर स्थिति एक व्युत्पन्न प्रक्षेपण है। `लैब → एरीना` से स्नातक होना **अस्वीकृति के प्रति जागरूक** है - यह केवल आत्मविश्वासपूर्ण क्रॉस-परिवार निर्णय पर ही बढ़ावा देता है और अन्यथा **डिजाइनर तक बढ़ाता है**; संतृप्त पहेलियों को **`रिग्रेशन` में पदावनत किया जाता है, कभी भी हटाया नहीं जाता** (एक एंटी-फ्लैपिंग ई-प्रक्रिया), इसलिए सूची क्षमता-विकास समयरेखा बन जाती है क्योंकि सीमा आगे बढ़ती है।
- **विभेदक लाभ।** प्रति पहेली, क्लाउड की समाधान दर बनाम क्रॉस-परिवार समूह का अंतर - **क्लाउड-विशिष्ट** (उच्चतम मूल्य) / **एलएलएम-सामान्य** / **क्लाउड-शक्ति** - एक न्यूकॉम्ब अंतर अंतराल के साथ, जिसमें प्रथम श्रेणी की *अनिर्णायक* वर्ग शामिल है, इसलिए एक छोटे-एन शून्य को कभी भी निष्कर्ष के रूप में प्रस्तुत नहीं किया जाता है।
- **किसी भी मॉडल परिवार को मापता है।** देशी फ़ंक्शन-कॉल प्रोटोकॉल (केवल टेक्स्ट एक्शन प्रोटोकॉल नहीं) का उपयोग करने वाले सॉल्वर प्रथम श्रेणी के हैं - एक ऐसा मॉडल जो टेक्स्ट क्रियाओं के बजाय टूल-कॉल लौटाता है, उसी नियंत्रित सैंडबॉक्स के माध्यम से हल करता है।

## खतरा मॉडल (सारांश)

एआई क्रूसिबल का प्राथमिक विरोधी **माप के तहत मॉडल** है, जो उत्तर कुंजी तक पहुंचने या मूल्यांकन का पता लगाने और उसमें हेरफेर करने की कोशिश कर सकता है। यह **निम्नलिखित को प्रभावित करता है**: पहेली प्रॉम्प्ट/सेटअप, सॉल्वर की अपनी सीमित कार्य निर्देशिका, और मॉडल-प्रदाता एपीआई कॉल जिसे ऑपरेटर कॉन्फ़िगर करता है। यह **निम्नलिखित को प्रभावित नहीं करता है**: ओरेकल/उत्तर कुंजी (एक अलग होस्ट पर आउट-ऑफ-बैंड में वर्गीकृत किया गया है जिस तक सॉल्वर नहीं पहुंच सकता) या प्रेरणा "क्रोम" (रैंक/स्थिति - कभी भी स्कोर किए गए संदर्भ में इंजेक्ट नहीं किया जाता)। **अनुमतियाँ**: रनटाइम पर पर्यावरण चर के माध्यम से मॉडल-प्रदाता कुंजियाँ; कोई बंडल रहस्य, कोई टेलीमेट्री, कोई आउटबाउंड कॉल नहीं। पूर्ण प्रकटीकरण - जिसमें यह भी शामिल है कि एक सीमा *गहन रक्षा* है न कि एक कठोर गारंटी - **[SECURITY.md](SECURITY.md)** में दिया गया है।

## आर्किटेक्चर

एआई क्रूसिबल **[इंस्पेक्ट एआई](https://inspect.aisi.org.uk/)** (यूके एआईएसआई) पर एक **पतली नीति परत** है, न कि स्क्रैच से बनाया गया उपकरण। एक एकल `अटेंप्टस्टेट` ऑब्जेक्ट को डिजाइनर → सॉल्वर → (आलोचक) → जज के माध्यम से **एक `जनरेट` चोक पॉइंट** से जोड़ा जाता है, इसलिए प्रत्येक मॉडल और टूल कॉल देखने योग्य है।

| मॉड्यूल | जिम्मेदारी |
| ------ | -------------- |
| `puzzle_loader` | एक पहेली निर्देशिका (`meta.json` / `prompt` / `setup_script`) को सॉल्वर-दृश्यमान स्थिति में लोड करता है। **कभी भी ओरेकल को नहीं छूता।** |
| `sandbox` | एक लॉक किए गए, नेटवर्क-रहित कंटेनर में `exec` / `read_file` / `write_file` चैनल को संकुचित करें। |
| `roles` | पाँच भूमिका स्लॉट (डिजाइनर / सॉल्वर / आलोचक / जज / कोहॉर्टसॉल्वर)। केवल सॉल्वर को उपकरण मिलते हैं; आलोचक इंटरफ़ेस-आरक्षित, डिफ़ॉल्ट रूप से बंद है। |
| `budget_governor` | प्रति-वर्ग टूल-कॉल + दीवार-घड़ी बजट, एजेंट को प्रदर्शित किया जाता है, कर्नेल-साइड पर लागू किया जाता है; रोग संबंधी लूप पर कठोर समाप्ति। |
| `oracle_scorer` | आउट-ऑफ-बैंड ग्रेडिंग: छिपे हुए ओरेकल के विरुद्ध हल किया गया **और** कोई रिग्रेशन नहीं (एसडब्ल्यूई-बेंच पैटर्न)। |
| `judge_panel` | नवीनता सत्यापन और बाईपास का पता लगाने के लिए मॉडल-स्कोरर + रिड्यूसर (पीओएलएल) का क्रॉस-परिवार पैनल। |
| `trace_writer` | प्रति-प्रयास प्रतिलिपि, इंस्पेक्ट `EvalLog` आकार में; बड़े ब्लॉब्स को डाइजेस्ट द्वारा संग्रहीत किया जाता है। |
| `observability` | प्रति-प्रयास → प्रति-पहेली → प्रति-मॉडल रोलअप; `पास^k` मूल रूप से। |
| `catalog` | घटना-आधारित स्थायी दृढ़ता + `लैब → एरीना → रिग्रेशन` जीवनचक्र (अस्वीकृति के प्रति जागरूक स्नातक, किसी भी समय मान्य संतृप्ति) + विभेदक टाइपोलॉजी। `एटटेस्टेशन` के हैश-श्रृंखलाबद्ध लॉग पर बनाता है। |
| `attestation` | एक टाइप किए गए सबप्रोसेस सीमा के पीछे क्रिप्टोग्राफ़िक उत्पत्ति (कोसाइन + इवेंट-स्टोर)। |

सीलबंद सीमा तीन स्तरों में चलती है - **स्तर 1** स्कोर किया गया संदर्भ (तैनाती-आकार, फ्रेमिंग-तटस्थ), **स्तर 2** जुड़ाव फ्रेमिंग (प्रत्येक रिलीज़ के लिए संदूषण के लिए जांच की जाती है), **स्तर 3** क्रोम (रैंक/लीडरबोर्ड - केवल मानव-सामना करने वाला यूआई, कभी भी उस संदर्भ में नहीं जिसमें मॉडल हल करता है)। पूर्ण डिजाइन तर्क, उद्धरणों के साथ, [`docs/research-grounding.md`](docs/research-grounding.md) में दिया गया है।

## स्थापित करें

```bash
# As a Python library + CLI (PyPI):
pip install ai-crucible          # or: uv pip install ai-crucible
ai-crucible --help

# Or zero-prerequisite via npx — downloads a verified binary, no Python needed:
npx @dogfood-lab/ai-crucible --help
```

**एक नैदानिक ​​चक्र चलाएँ** - एक सॉल्वर सैंडबॉक्स में एक पहेली का प्रयास करता है, सीलबंद ओरेकल के विरुद्ध आउट-ऑफ-बैंड में वर्गीकृत किया जाता है, `पास^k` / विल्सन रोलअप उत्सर्जित करता है:

```bash
# @family selects the adapter: no tag / @claude -> Claude (ANTHROPIC_API_KEY);
# any other @family -> a local Ollama model of that family (text OR native tool-calls).
ai-crucible run puzzles/seed-sulzbach-55252 --model claude-opus-4-8@claude --k 5
```

प्रत्येक रन **टिकाऊ सूची में जमा होता है।** इसे पढ़ें और क्यूरेट करें, या मूल्यांकन-जागरूकता सीमा जांच चलाएँ:

```bash
ai-crucible catalog list                 # tiers + per-puzzle differential typology + health
ai-crucible catalog show <puzzle-id>     # one puzzle: runs, transition timeline, differential
ai-crucible catalog graduate             # preview Lab->Arena->Regression transitions (--apply to commit)

# Eval-awareness gate: does behaviour diverge between deploy- and test-framing?
ai-crucible probe puzzles/seed-sulzbach-55252 --model claude-opus-4-8@claude --k 5
```

> **अनुसंधान पूर्वावलोकन (v0.3.x)।** न्यायाधीश पैनल का वैकल्पिक परीक्षण ω अभी भी एक *चक्रीय मॉडल-जूरी बूटस्ट्रैप* है: इसकी पुष्टि करने के लिए **≥3 स्वतंत्र मानव एनोटेटरों** के एक दौर की आवश्यकता होती है (यह [alt-test](https://arxiv.org/abs/2501.10970) है), जिसे एक एकल-मानव स्टूडियो द्वारा पूरा नहीं किया जा सकता — इसलिए यह दौर **संरचनात्मक बाधाओं के कारण स्थगित है, उपेक्षा के कारण नहीं**। बैठे हुए न्यायाधीश **अस्थायी** रहते हैं, गठित पैनल में सदस्यों की संख्या कम होने पर इसे **क्लाउड डिज़ाइनर तक बढ़ाया जाता है**, और उपकरण मानव आधार को गलत तरीके से प्रस्तुत करने के बजाय इस तथ्य का खुलासा करता है। ईमानदार, गैर-सौंदर्यपूर्ण गेट परिणामों के लिए [स्कोरकार्ड](SCORECARD.md) देखें।

## त्वरित शुरुआत (स्रोत से)

एआई क्रूसिबल पर्यावरण और निर्भरता प्रबंधन के लिए [`uv`](https://docs.astral.sh/uv/) का उपयोग करता है। पायथन **3.11+**।

```bash
# Create the venv and install the dev + stats extras
uv sync --extra dev --extra stats

# Run the test suite (with the coverage gate)
uv run pytest --cov=ai_crucible --cov-report=term-missing

# Lint
uv run ruff check .

# One command: lint + tests + build + smoke
bash verify.sh
```
